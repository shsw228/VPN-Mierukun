# 仕様書

## 目的
VPN の接続状態を macOS 上で即座に認識できるようにし、接続漏れや切断に気づきやすくする。

## MVP のゴール
- ユーザーが 1 つの VPN サービスを選択できる
- 選択した VPN の状態を数秒以内に画面周囲の色で判別できる
- メニューバーから現在状態、監視対象、オーバーレイの有効/無効を操作できる
- アプリ再起動後も設定が保持される

## 想定ユーザー
- 業務で VPN 接続が必須な macOS ユーザー
- リモートワーク中に VPN の接続状態を常時把握したいユーザー
- VPN 切断時の見落としを減らしたいユーザー

## 想定ユースケース
- 仕事開始時に VPN 接続済みかを一瞬で確認する
- 作業中に VPN が切断されたら即座に気づく
- フルスクリーン作業中でも、通知ではなく画面端の色で接続状態を把握する
- マルチディスプレイ環境でも各画面で状態を見失わない

## 機能要件
- VPN の接続状態を定期的に監視できること
- 接続状態に応じて画面周囲のオーバーレイ色を切り替えられること
- メニューバーから現在状態を確認できること
- オーバーレイ表示の有効/無効を切り替えられること
- 状態ごとの表示色や線幅を設定できること
- 監視対象 VPN サービスを選択できること
- 画面構成の変化を検知し、接続中ディスプレイに追従できること

## 状態定義
- `connected`: VPN が接続中。表示色は緑系
- `disconnected`: VPN が未接続。表示色は赤系
- `transitioning`: 接続中断や再接続中などの遷移状態。表示色は黄系
- `unknown`: 判定不能。表示色は灰色系

## MVP 対象外
- 複数 VPN サービスの同時監視
- 接続操作の実行機能
- 通知センター連携やサウンド通知
- 履歴グラフや通信量表示
- App Store 配布対応の詰め

## 非機能要件
- 常駐しても CPU・メモリ消費が過大にならないこと
- 通常作業の邪魔にならない表示であること
- マルチディスプレイ環境でも破綻しにくいこと
- ネットワーク状態の変化に対して短時間で追従すること
- 起動直後 3 秒以内に前回設定を読み込み、監視を開始できること

## 配布要件
- GitHub Releases でアプリの ZIP を公開できること
- Homebrew tap の `cask` からインストールできること

## 画面/操作仕様
- メニューバー常駐アプリとして起動する
- 接続中は画面周囲を緑系、未接続は赤系、遷移中は黄系、判定不能は灰色系で表示する
- オーバーレイはクリック操作を阻害しない見た目と挙動を目指す
- オーバーレイは各ディスプレイの四辺に細い帯として表示する
- メニューバーでは状態表示、監視対象表示、手動更新、設定表示、終了が行える
- 設定画面では監視対象 VPN、色、表示幅、起動時動作を変更できるようにする

## データ仕様
- 外部サーバーへのデータ送信は行わない想定
- ローカルに保持するのは設定情報のみとする
- 保存対象:
  - 監視対象 VPN 名
  - 状態別カラー
  - オーバーレイ幅
  - オーバーレイ有効フラグ
  - 起動時に監視を開始するか

## 受け入れ条件
- VPN 接続中に画面周囲が緑系表示へ切り替わる
- VPN 未接続時に画面周囲が赤系表示へ切り替わる
- 接続遷移中または判定不能時に、定義済みの代替色へ切り替わる
- オーバーレイがマウス操作や通常のウィンドウ操作を妨げない
- 監視対象と色設定がアプリ再起動後も保持される

## 制約・前提
- macOS のネットワーク情報取得方法に依存する
- VPN の種類によっては状態取得方法の差異を吸収する必要がある
- 画面最前面のオーバーレイ表示には macOS の制約や権限を確認する必要がある
- App Store 配布を目指す場合は利用 API や権限周りの制約を別途確認する
- 初期実装では「ユーザーが macOS に登録済みの VPN サービスを 1 つ選ぶ」前提とする

## 今後の検討事項
- VPN 状態取得の実装を `scutil` ベースで開始するか、別 API に切り替えるか
- 切断時の通知やサウンド連携の要否
- 複数 VPN プロファイル対応の優先度
- ログ出力や履歴表示の必要性
- 自動起動設定の実装方法
- sandbox / App Store 配布を視野に入れた API 制約整理
