# AGENTS.md

## 目的
- VPN 接続状態を可視化する macOS アプリを安全かつ段階的に実装する
- 変更理由と影響範囲を明確にし、仕様と実装の差分を小さく保つ

## スコープ
- 上位のシステム指示・ユーザー指示・リポジトリ外の共通 AGENTS 指示を優先する
- このファイルは本リポジトリ固有の運用ルールのみを補足する

## 作業開始時チェック
- まずリポジトリ構成と既存ファイルを確認する
- 既存の未コミット変更がある場合は消さずに作業方針を合わせる
- 仕様変更を伴う実装では `docs/specification.md` を先に確認する

## 実装ルール
- 破壊的操作は事前確認なしで実行しない
- 既存の未コミット変更は巻き戻さない
- 既存の命名規則・UI 方針・技術選定があればそれを優先する
- 画面オーバーレイ、権限、VPN 状態取得のような影響の大きい変更は段階的に進める

## ドキュメント運用
- 機能追加や仕様変更を行った場合は `docs/specification.md` を更新する
- セットアップや開発手順が固まったら `README.md` を更新する

## テスト/検証方針
- 実装時は可能な範囲で再現手順と確認結果を残す
- VPN 接続状態の変化、マルチディスプレイ、オーバーレイの操作阻害有無を優先確認項目とする
- 実行できない検証がある場合は理由を明記する

## コミット/PR方針
- コミットは依頼がある場合のみ行う
- コミット・PR の形式や文体は上位指示または既存規約に従う
- 仕様に影響する変更は PR 説明でも利用者影響を簡潔に示す
